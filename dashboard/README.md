# Dashboard (Web) “пульт управления и телеметрия”

## Цель
Веб-интерфейс показывает симуляцию в удобном виде:
- лента событий в реальном времени
- состояния агентов (эмоции, планы)
- граф отношений
- управление: событие в мир, сообщение агенту, скорость времени
- инспектор агента (память + последние события)

Dashboard доступен на `/` через gateway.

## MVP (обязательный)
### 1) Realtime лента событий
- подключение к WS: `ws://<host>/ws/stream`
- отображение последних 200–300 событий
- фильтр по агенту (опционально, если успеем)

### 2) Панель агентов
- карточки агентов: имя, настроение (mood_label), текущий план
- кнопка “открыть инспектор”

### 3) Граф отношений
- узлы: агенты
- ребра: отношения (value -100..+100)
- визуально: толщина/прозрачность/цвет по модулю и знаку value

### 4) Управление
- POST `/api/control/event` — добавить событие
- POST `/api/control/message` — сообщение агенту
- POST `/api/control/speed` — скорость симуляции

### 5) Инспектор агента
- GET `/api/agents/{id}`
- показывает: traits, mood, current_plan, key_memories, recent_events

## Режим обновления
- Стартовый снапшот: `GET /api/state`
- Реалтайм: WebSocket `/ws/stream`
Если WS сложен — допускается polling `/api/state` раз в 1–2 сек, но WS предпочтительнее.

## Definition of Done
- открывается `/`
- видны агенты, лента, граф
- интерфейс управляет симуляцией (событие/сообщение/скорость)
- данные обновляются в реальном времени
